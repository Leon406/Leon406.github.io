#git 工作流使用规范

0.初始化 一般只执行一次

0.1更换服务器ip 重新连接

1.切换分支dev 开发 ( 从 步骤一到步骤五)

2.开发完成切回master merge

​ 不建议使用merge会自动交叉合并

3.将本地master内容推送到服务器 develop分支 (不要省略 :develop )

​ 本地commit 3到5次 或者写了100~200行代码时,再执行

​ 代码运行没有错误再提交

3.1无法获取时 从服务器分支获取最新数据

3.2有冲突,解决后提交,根据提示CONFLICT 找到对应文件并且修改

3.3重新提交

4.提交成功后,删除dev分支

5.切换分支继续开发

其他:

重置远端服务器地址
